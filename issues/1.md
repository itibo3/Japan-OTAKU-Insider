# 🗂️ JOIA 改善実装ロードマップ＆仕様書

> **使い方：** タスクごとにコメントに分割。1コメントずつコピペしてCopilot/Claudeに投げる。
> **完了したらIssueをクローズ or 削除してOK**

---

## 🚀 全体ワークフロー（完成形）

```
【毎日 19:00 JST 自動実行】

① RSS取得（rss_fetch.py）
   └ キーワードフィルター付き（TASK1で追加）
     ↓
② Perplexity検索（perplexity_search.py）
   └ cafe / vtuber / other の3カテゴリのみ推奨
     ↓
③ DeepL翻訳（translate_with_deepl.py）
     ↓
④ entries.jsonに追加（add_entry.py）
     ↓
⑤ ソート・サムネイル補完・ステータス更新
     ↓
⑥ コミット・プッシュ → GitHub Pages自動デプロイ
     ↓
⑦ X自動投稿（post_to_x.py）
     ↓
⑧ 品質チェック自動実行（quality_check.py）← TASK3で追加
   └ Perplexity記事のみ対象
   └ 未翻訳タグ残り・短すぎ・NGワードを自動削除

【毎週月曜 19:00 JST 自動実行】

⑨ 週間レポート生成（weekly_report.py）← TASK4で追加
   └ Perplexity記事の生存率をカテゴリ別に集計
   └ GitHubのIssueに自動投稿

【手動実行（必要な時だけ）】

⑩ 記事削除フォーム（delete-entry.yml）← TASK2で追加
   └ ActionsタブからIDを入力して実行
   └ entries.jsonから該当記事を削除してプッシュ
```

## ✅ 実装後の確認手順

```
TASK1確認
→ Actions → Daily Update → Run workflow
→ ログで「SKIP (NG keyword)」が出ればOK

TASK2確認
→ Actions → Delete Entry → Run workflow
→ entry_idに適当なIDを入力して実行
→ entries.jsonから消えていればOK

TASK3確認
→ Daily Update完了後にquality_checkジョブが
  連鎖実行されればOK

TASK4確認
→ Actions → Weekly Report → Run workflow
→ IssuesにレポートIssueが投稿されればOK
```
